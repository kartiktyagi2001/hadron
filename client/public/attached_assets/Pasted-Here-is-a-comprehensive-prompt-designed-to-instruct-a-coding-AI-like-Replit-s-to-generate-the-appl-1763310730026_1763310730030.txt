### **Key Component Specifications**

**1. `HeroBanner.tsx` (Homepage)**

  * **Goal:** Create the "Exicom-inspired" hero.
  * **Layout:** Full-width section with centered text.
  * **Background:** Use a subtle, low-opacity background pattern (like a dot grid or light schematic) to give the "electronics touch."
  * **Content:** A main headline (H1), a sub-headline (p), and a CTA button.
  * **Animation (Framer Motion):**
      * The H1, paragraph, and button should each fade in and slide up slightly on page load.
      * Use `variants` for `opacity: 0, y: 20` animating to `opacity: 1, y: 0`.
      * Stagger the animations (e.g., delay 0.1s, 0.3s, 0.5s).
  * **Components:** Use `shadcn/ui` **Button** for the CTA.

**2. `Navbar.tsx`**

  * **Goal:** Clean, professional, and responsive navigation.
  * **Layout:**
      * **Desktop:** Logo on the left, navigation links (`Home`, `About`, `Products`, `Contact`) in the center, and a "Get in Touch" Button on the right.
      * **Mobile:** Logo on the left, Hamburger Menu icon on the right.
  * **Components:** Use `shadcn/ui` **Sheet** (or Drawer) for the mobile menu.
  * **Styling:** Should have a subtle bottom border or shadow to separate it from the page content.

**3. `ProductCard.tsx`**

  * **Goal:** A reusable card for the product grid.
  * **Components:** Use `shadcn/ui` **Card** component.
  * **Content:**
      * `CardHeader`: Product Image (use Next.js `<Image>`).
      * `CardContent`: Product Title (`<h3>`) and Short Description (`<p>`).
      * `CardFooter`: A "View Details" button.
  * **Animation (Framer Motion):**
      * On hover (`whileHover`), the card should lift slightly (e.g., `scale: 1.03` or `y: -5`).

**4. `ProductGrid.tsx` (`/products/page.tsx`)**

  * **Goal:** Display all products from `products.json`.
  * **Logic:** Fetch and map the JSON data.
  * **Layout:** A responsive CSS grid (e.g., `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`).
  * **Animation (Framer Motion):**
      * Wrap the grid container in a `motion.div`.
      * Use `staggerChildren` on the container so each `ProductCard` fades in one after the other.

**5. `ProductDetail.tsx` (`/products/[slug]/page.tsx`)**

  * **Goal:** Display a single product's details.
  * **Logic:** This is a dynamic page. It must use the `slug` from the URL to find the correct product in `products.json`.
  * **Layout:** A 2-column layout (Image on the left, details on the right).
  * **Content:**
      * Product Title
      * Full Description
      * A list or table of the `specifications` object.
      * An "Inquire Now" button (links to the Contact page).
